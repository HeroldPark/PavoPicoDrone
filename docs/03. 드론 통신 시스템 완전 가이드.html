🔄 신호 흐름 상세 분석

📡 제어 신호 경로
조종기 스틱 조작 ↓ EdgeTX 펌웨어 처리 ↓ 2.4GHz RF 모듈 송신 ↓ 공중 전파 (2.4GHz) ↓ 드론의 SBUS 수신기 ↓ SBUS 프로토콜 디코딩 ↓ Arduino Nano ESP32 ↓ PID 제어 알고리즘 ↓ 모터 PWM 출력
📊 제어 신호 특성
  • 주파수: 2.4GHz ISM 대역
  • 프로토콜: SBUS (100kbps)
  • 지연시간: ~20-30ms
  • 범위: 1-2km (가시거리)
  • 채널: 16채널 동시 전송
📹 영상 신호 경로
카메라 영상 촬영 ↓ Avatar HD V2 인코딩 ↓ H.264/H.265 압축 ↓ 5.8GHz 송신 ↓ 공중 전파 (5.8GHz) ↓ 고글의 수신기 ↓ 영상 디코딩 ↓ 디스플레이 출력 ↓ 조종자 시야
📊 영상 신호 특성
  • 주파수: 5.8GHz (한국 불법)
  • 해상도: 1080p60fps
  • 지연시간: ~25-40ms
  • 범위: 500m-4km
  • 코덱: H.265 (HEVC)
📊 텔레메트리 경로
센서 데이터 수집 ↓ Arduino 데이터 처리 ↓ MSP 프로토콜 패킷 ↓ SBUS 수신기 (업링크) ↓ 2.4GHz 역방향 전송 ↓ 조종기 수신 ↓ EdgeTX 디스플레이 ↓ 조종자 모니터링
📊 텔레메트리 특성
  • 주파수: 2.4GHz (역방향)
  • 프로토콜: MSP, SmartPort
  • 업데이트: 1-10Hz
  • 데이터: 배터리, GPS, 자세
  • 범위: 제어신호와 동일

🔗 바인딩 과정 단계별 가이드

📡 조종기 ↔ 수신기 바인딩

1

조종기 설정

  • 조종기 전원 ON
  • Model → [새 모델] 생성
  • RF 프로토콜 선택 (FrSky D16, ExpressLRS 등)
  • Region: EU LBT (한국 기준)
2

수신기 바인딩 모드

  • 드론 전원 OFF 상태
  • 수신기의 BIND 버튼 누르면서 드론 전원 ON
  • 수신기 LED가 빠르게 깜빡임 → 바인딩 모드
3

조종기에서 바인딩 실행

  • 조종기: Model → Bind → [실행]
  • 조종기가 바인딩 신호 송출
  • 수신기 LED가 고정 점등되면 성공
4

연결 확인

  • 조종기 스틱 조작
  • 수신기 LED 상태 확인
  • Arduino 시리얼 모니터로 SBUS 데이터 확인
  • 모터 테스트 (프로펠러 제거 상태)

📹 VTX ↔ 고글 바인딩

1

VTX 설정

  • 드론 전원 ON
  • VTX가 자동으로 송신 시작
  • 기본 채널: Race1~Race8 중 하나
  • 전력: 25mW (기본값)
2

고글 스캔

  • 고글 전원 ON
  • Auto Scan 실행
  • 5.8GHz 대역 전체 스캔
  • 신호 감지 시 자동 선택
3

HD 링크 설정

  • 고글에서 VTX 감지
  • HD Link 메뉴에서 바인딩
  • VTX와 고글 ID 매칭
  • 1080p60 모드 설정
4

영상 확인

  • 카메라 영상이 고글에 표시
  • OSD 정보 오버레이
  • 지연시간 체크 (~25ms)
  • 녹화 기능 테스트

🔧 통신 프로토콜 상세

SBUS 프로토콜

📋 SBUS 데이터 구조

  • 프레임 크기: 25바이트
  • 헤더: 0x0F
  • 채널 데이터: 16채널 × 11비트
  • 플래그: Failsafe, Frame Lost
  • 엔더: 0x00
0x0F | CH1-16 데이터 | 플래그 | 0x00 | 22바이트 | 1바이트 | 헤더 엔더
⚙️ Arduino 설정 
Serial2.begin(100000, SERIAL_8E2, 4, -1, true);
  • 100000 bps
  • 8비트, Even 패리티, 2 정지비트
  • 신호 반전 필요 (true)
MSP 프로토콜

📋 MSP 메시지 구조

  • 헤더: $M< 또는 $M>
  • 크기: 데이터 길이
  • 명령: MSP_ATTITUDE 등
  • 데이터: 센서 값
  • 체크섬: XOR 검증
$M< | 길이 | 명령 | 데이터 | 체크섬
📊 주요 MSP 메시지
  • MSP_ATTITUDE: Roll/Pitch/Yaw
  • MSP_ANALOG: 배터리 전압
  • MSP_RAW_GPS: GPS 정보
  • MSP_STATUS: 시스템 상태
Avatar HD 프로토콜

📋 Avatar HD 통신 구조

  • 영상 코덱: H.265 (HEVC)
  • 해상도: 1080p @ 60fps
  • 비트레이트: 10-50 Mbps (적응형)
  • 지연시간: 25-40ms (글래스 투 글래스)
  • 주파수: 5.8GHz ISM 대역
  • 채널: 8개 채널 (Race1-8)
카메라 센서 (IMX708) ↓ ISP (이미지 신호 처리) ↓ H.265 하드웨어 인코더 ↓ RF 모듈 (5.8GHz) ↓ 안테나 송출 ↓ 고글 수신 안테나 ↓ RF 복조 및 디코딩 ↓ 디스플레이 출력
📺 영상 품질 모드
  • 고화질: 1080p60 @ 25Mbps
  • 표준: 1080p30 @ 15Mbps
  • 저지연: 720p120 @ 20Mbps
  • 장거리: 720p60 @ 8Mbps

🔧 Avatar HD V2 설정 명령어

AT+POWER=2 // 출력 레벨 (0=25mW, 1=200mW, 2=800mW) AT+FREQ=5800 // 주파수 설정 (MHz) AT+CHAN=1 // 채널 번호 (1-8) AT+BITRATE=25 // 비트레이트 (Mbps) AT+MODE=0 // 화질 모드 (0=고화질, 1=저지연)

📶 주파수 대역 및 채널 구조

✅ 2.4GHz ISM 대역 (합법)

📡 2.4GHz 상세 스펙

  • 주파수 범위: 2400-2483.5 MHz
  • 채널 수: 79개 (1MHz 간격)
  • 최대 출력: 100mW (20dBm)
  • 변조 방식: FHSS, DSSS
  • 허가: 자유 사용 (KC 인증)
🎮 2.4GHz 프로토콜 종류
  • FrSky D16: 16채널, ACCST
  • ExpressLRS: 1000Hz, 오픈소스
  • Crossfire: 원거리, 150Hz
  • FlySky AFHDS: 저가형

📊 2.4GHz 채널 할당

Ch 1: 2412 MHz Ch 6: 2437 MHz Ch 11: 2462 MHz Ch 2: 2417 MHz Ch 7: 2442 MHz Ch 12: 2467 MHz Ch 3: 2422 MHz Ch 8: 2447 MHz Ch 13: 2472 MHz Ch 4: 2427 MHz Ch 9: 2452 MHz Ch 14: 2484 MHz Ch 5: 2432 MHz Ch 10: 2457 MHz (일본만)
❌ 5.8GHz ISM 대역 (한국 불법)

⚠️ 5.8GHz 상세 스펙

  • 주파수 범위: 5725-5875 MHz
  • 채널 수: 40개 (Race, Band A-E)
  • 출력: 25mW-2W
  • 변조: FM, 디지털
  • 한국 상태: 전파법 위반
📺 5.8GHz 채널 구조
  • Race Band: R1-R8 (5658-5917MHz)
  • Band A: A1-A8 (5865-5945MHz)
  • Band B: B1-B8 (5733-5813MHz)
  • Band E: E1-E8 (5705-5885MHz)

🚨 한국 전파법 위반 사항

  • 벌금: 최대 300만원
  • 몰수: 장비 압수 가능
  • 간섭: 기상레이더 방해
  • 탐지: 전파감시소 자동 탐지

🔧 Avatar HD V2 상세 설정 가이드

⚙️ 초기 설정 과정

1

하드웨어 연결

  • 전원: 3.3V-5V 입력 (Arduino 3.3V 권장)
  • UART: TX→D17, RX→D16 (115200bps)
  • 카메라: 전용 리본 케이블 연결
  • 안테나: MMCX 커넥터 (LHCP 권장)
2

펌웨어 설정

  • 최신 펌웨어: Walksnail 공식 사이트 다운로드
  • 업그레이드: micro-SD 카드 이용
  • 설정 초기화: 팩토리 리셋 수행
  • 지역 설정: CE (유럽) 또는 FCC (미국)
3

채널 및 전력 설정

AT 명령어로 설정

// Arduino에서 VTX 설정 void setupVTX() { Serial1.begin(115200); delay(1000); // 채널 설정 (Race1 = 5658MHz) Serial1.println("AT+FREQ=5658"); delay(100); // 출력 전력 (0=25mW, 1=200mW, 2=800mW) Serial1.println("AT+POWER=0"); // 25mW로 시작 delay(100); // 화질 모드 (0=고화질, 1=저지연) Serial1.println("AT+MODE=0"); delay(100); // 비트레이트 설정 Serial1.println("AT+BITRATE=15"); delay(100); Serial.println("VTX 설정 완료"); }
4

OSD 설정

  • MSP 연결: Arduino → VTX UART
  • 배터리 전압: 좌상단 표시
  • 비행 시간: 우상단 표시
  • 인공수평선: 중앙 표시

MSP OSD 데이터 전송

void sendOSDData() { // 배터리 전압 전송 sendMSPBattery(batteryVoltage); delay(10); // 자세 정보 전송 sendMSPAttitude(roll, pitch, yaw); delay(10); // GPS 정보 전송 (더미) sendMSPGPS(); delay(10); } // 50Hz로 호출 void loop() { static unsigned long lastOSD = 0; if (millis() - lastOSD > 20) { sendOSDData(); lastOSD = millis(); } }

🥽 Walksnail Goggles X 연결 설정

🔍 채널 스캔 및 바인딩

자동 스캔 과정

  • RSSI: -60dBm 이상 권장
  • SNR: 15dB 이상 양호
  • 패킷 로스: 1% 이하
  • 지연시간: 25-35ms 정상
⚙️ 화질 및 디스플레이 설정

최적 화질 설정

  • 해상도: 1080p60 (기본)
  • FOV: 120° (몰입감)
  • 밝기: 80% (배터리 절약)
  • 대비: 자동 조절
  • 채도: +10% (선명도)
🔋 배터리 최적화
  • 절전 모드: 5분 후 자동
  • 화면 밝기: 환경에 따라 조절
  • DVR 녹화: 필요시만 활성화
  • Wi-Fi: 사용 안할 때 끄기
